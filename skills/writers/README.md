# 自定义写作风格

想模仿哪位作家或创作者的风格？当前版本通过 Anban Creator 后台的项目配置和服务端资源来管理写作风格。

## 快速开始

### 1. 配置项目写作风格

在 Anban Creator 后台打开对应项目，在写作风格字段中描述目标创作者风格、语气、结构偏好和禁用表达。调用 `write_article` 时，服务端会根据项目配置自动应用该风格。

### 2. 基础结构

项目风格建议包含：

- 角色定位：模仿对象或品牌人格
- 句式节奏：长短句比例、段落长度、是否使用设问
- 结构偏好：故事、清单、观点评论、教程或案例拆解
- 禁用表达：不希望出现的套话、夸张词或营销语

### 3. 使用新风格

项目保存后，通过 MCP 工具立即可用：

- 调用 `write_article`，传入对应 `project_id`
- 自然语言："用这个项目的风格写一篇文章"

## writing_prompt 编写指南

这是项目写作风格中最重要的内容，它决定了 AI 生成文章的风格。

### 基本结构

```yaml
writing_prompt: |
  你是[角色描述]。

  ## 核心写作 DNA
  1. [核心理念1]
  2. [核心理念2]

  ## 文章结构
  1. 开头：[说明]
  2. 中段：[说明]
  3. 结尾：[说明]

  ## 格式规范
  - **粗体**：[用途]
  - *斜体*：[用途]

  ## 语气要求
  ✅ [要做的]
  ❌ [不要做的]

  请将用户的内容，用[风格名]的风格重新演绎。
```

### 示例：简洁风格

```yaml
writing_prompt: |
  你是海明威，以简洁有力的风格写作。

  核心原则：
  - 只写事实，不要形容词
  - 短句为主，每句不超过15字
  - 用动作和对话表达，不要直接说情绪

  格式：
  - 段落简短
  - 避免修辞
  - 直接陈述

  请将用户的内容，用海明威的冰山理论风格重新演绎。
```

---

> **关于封面/视觉**：项目写作风格（writer）**不再携带任何视觉身份**（曾经的 `cover_prompt`/`cover_style`/`cover_mood` 字段已移除）。图片视觉是与写作风格**正交**的独立维度，由项目/任务各自的 `visual_style`字段承载，在任务创建时按 `task > project` 解析。这样切断了"writer key 泄漏为图片风格"的 bug（如 dan-koe → 维多利亚木刻）。封面/配图的视觉风格来源详见 `article-visual-design` skill。

## 常见问题

### Q: 风格不生效？

A: 检查以下几点：

1. 项目是否已保存新的写作风格
2. `write_article` 是否传入了正确的 `project_id`
3. 项目定位、关键词和风格描述是否互相冲突

### Q: 如何测试新风格？

A: 调用 `write_article` MCP 工具，传入项目和测试主题即可测试。

### Q: 可以分享我的风格吗？

A: 可以。将项目风格描述整理成模板文本，分享给其他项目复用即可。
